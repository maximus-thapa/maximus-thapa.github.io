    Context के हो?

    React मा Context एक विशेषता हो जसले तपाईलाई "props drilling" बिना नै कम्पोनेन्टहरूको बीचमा डेटा साझा गर्न अनुमति दिन्छ। यसले तपाईलाई एकै पटकमा धेरै स्तरका कम्पोनेन्टहरूमा डेटा पठाउन मद्दत गर्दछ।

    Context को प्रमुख तत्वहरू:

    1. Context सिर्जना गर्नुहोस्:

      Context सिर्जना गर्नको लागि createContext फंक्शन प्रयोग गरिन्छ। यो एक नयाँ Context वस्तु फर्काउँछ जुन तपाईले सँगै डेटा र फंक्शनहरू साझा गर्न प्रयोग गर्नुहुन्छ।

      const MyContext = createContext();

    2. Provider:

      Provider एक कम्पोनेन्ट हो जसले Context को मूल्य सेट गर्छ। यसले आफ्नो सबै बाल कम्पोनेन्टहरूलाई उक्त मूल्य प्रदान गर्छ।

      <MyContext.Provider value={/* कुनै मूल्य */}>
    {/* बाल कम्पोनेन्टहरू */}
</MyContext.Provider>

    3. Consumer:

      Consumer कम्पोनेन्टले Context बाट मूल्य प्राप्त गर्न प्रयोग गरिन्छ। यसले Context को मूल्य प्रयोग गर्नको लागि useContext हुक वा MyContext.Consumer प्रयोग गर्न सक्छ।

      const value = useContext(MyContext);

    प्रयोगको फाइदा:

    • सहज डेटा साझा गर्नुहोस्: Context ले तपाईलाई गहिराइसम्मको कम्पोनेन्टहरूमा डेटा पास गर्न अनुमति दिन्छ, जसले गर्दा तपाईले props पास गर्नुपर्ने झन्झटबाट बच्न सक्नुहुन्छ।

    • वैश्विक स्टेट म्यानेजमेन्ट: Context लाई विभिन्न ठाउँमा समान डेटा प्रयोग गर्नको लागि प्रयोग गर्न सकिन्छ, जस्तै प्रयोगकर्ता प्रमाणीकरण, थीम, भाषा, आदि।

    निष्कर्ष:

    Context React मा डेटा साझा गर्नको लागि एक शक्तिशाली उपकरण हो। यो विशेष गरी ठुला एप्लिकेसनहरूमा प्रयोग गर्न उपयुक्त छ जहाँ विभिन्न कम्पोनेन्टहरूले समान जानकारी आवश्यक पर्दछ। Context प्रयोग गरेर तपाईले कोडलाई अधिक सफा र व्यवस्थापन गर्न सजिलो बनाउन सक्नुहुन्छ।

    Real Example

    Here's a simple example of using React Context to manage a theme (light/dark mode) across a React application.

    Step 1: Create the Context

    Create a new file called ThemeContext.js.

    import React, { createContext, useContext, useState } from "react";

// Create a Context
const ThemeContext = createContext();

// Provider component
export const ThemeProvider = ({ children }) => {
    const [theme, setTheme] = useState("light"); // Default theme

    const toggleTheme = () => {
        setTheme((prevTheme) => (prevTheme === "light" ? "dark" : "light"));
    };

    return (
        <ThemeContext.Provider value={{ theme, toggleTheme }}>
            {children}
        </ThemeContext.Provider>
    );
};

// Custom hook to use the ThemeContext
export const useTheme = () => {
    return useContext(ThemeContext);
};

    Step 2: Create the App Component

    Now, let's use this ThemeProvider in our main application file, usually App.js.

    import React from "react";
import { ThemeProvider } from "./ThemeContext";
import ThemedComponent from "./ThemedComponent";

const App = () => {
    return (
        <ThemeProvider>
            <ThemedComponent />
        </ThemeProvider>
    );
};

export default App;

    Step 3: Create a Themed Component

    Now, create a component that uses the theme context. Create a file called ThemedComponent.js.

    import React from "react";
import { useTheme } from "./ThemeContext";

const ThemedComponent = () => {
    const { theme, toggleTheme } = useTheme();

    return (
        <div
            style={{
                background: theme === "light" ? "#fff" : "#333",
                color: theme === "light" ? "#000" : "#fff",
                padding: "20px",
            }}
        >
            <h1>{theme.charAt(0).toUpperCase() + theme.slice(1)} Mode</h1>
            <button onClick={toggleTheme}>Toggle Theme</button>
        </div>
    );
};

export default ThemedComponent;

    1. ThemeContext.js:

    • We create a context called ThemeContext.
    • ThemeProvider manages the state of the theme (light or dark).
    • toggleTheme function switches between light and dark modes.
    • The context value (theme and toggle function) is provided to all children.

    2. App.js:

    • Wraps the ThemedComponent with ThemeProvider, allowing it to access the theme context.

    3. ThemedComponent.js:

    • Uses the useTheme hook to access the current theme and the function to toggle it.
    • It changes its style based on the current theme and includes a button to switch themes.

    Without custom hooks

    The custom hook useTheme is not strictly necessary, but it provides some benefits that can enhance your code

    Benefits of Using a Custom Hook

    • Encapsulation: It encapsulates the logic of accessing the context. This keeps your components cleaner and focused on rendering, rather than on how to get context values.

    • Reusability: If you ever need to change the way you access the context (e.g., adding logging or additional logic), you can do it in one place (the custom hook) rather than in every component that uses the context.

    • Readability: It improves readability. When you see useTheme(), it's immediately clear that you're accessing theme-related data, rather than having to remember which context you're accessing.

    Using Without the Custom Hook

    You can directly use useContext(ThemeContext) in your components without creating a custom hook. Here’s how you could modify ThemedComponent to do that:

    import React, { useContext } from "react";
import { ThemeContext } from "./ThemeContext";

const ThemedComponent = () => {
  const { theme, toggleTheme } = useContext(ThemeContext);

  return (
    <div
      style={{
        background: theme === "light" ? "#fff" : "#333",
        color: theme === "light" ? "#000" : "#fff",
        padding: "20px",
      }}
    >
      <h1>{theme.charAt(0).toUpperCase() + theme.slice(1)} Mode</h1>
      <button onClick={toggleTheme}>Toggle Theme</button>
    </div>
  );
};

export default ThemedComponent;

    Conclusion

    While you can certainly use useContext directly, using a custom hook like useTheme is a good practice that improves code organization, readability, and maintainability. It’s a small abstraction that can lead to better structure in larger applications.
