    Example

    Sure! Here’s a complete example that includes a higher-order function and a callback function. This example simulates a simple game where you can choose different actions to play with.

    Complete Example

    // Higher-order function
function toyBox(action) {
  console.log("Let's play with the toy!");
  action(); // Using the callback function
}

// Callback function 1
function jump() {
  console.log("Jumping up and down!");
}

// Callback function 2
function spin() {
  console.log("Spinning around!");
}

// Callback function 3
function dance() {
  console.log("Dancing to the music!");
}

// Using the toy box with different toys (callback functions)
toyBox(jump); // Will call the jump function
toyBox(spin); // Will call the spin function
toyBox(dance); // Will call the dance function

    What Happens Here

    1. Higher-Order Function: toyBox is the higher-order function that accepts another function (action) as an argument.
    2. Callback Functions: jump, spin, and dance are the callback functions that define different actions.
    Output

    When you run this code, you’ll see:

    Let's play with the toy!
Jumping up and down!
Let's play with the toy!
Spinning around!
Let's play with the toy!
Dancing to the music!

    Summary

    • The toyBox function can take any of the action functions (jump, spin, or dance) and use them.
    • Each action is a callback function that gets executed inside the higher-order function.

    Callback Function

    Standalone Function vs. Callback Function

    1. Standalone Function:

      • The function dance on its own is just a regular function. It doesn't do anything until you call it.
      function dance() {
  console.log("Dancing to the music!");
}

    2. Callback Function:

      • A function becomes a callback when it is passed as an argument to another function and called within that function. For example:
      function toyBox(action) {
  console.log("Let's play with the toy!");
  action(); // Here, action is calling the callback function
}

toyBox(dance); // Now, dance is used as a callback function

    Summary

    • Standalone Function: Just a regular function that can be called independently.
    • Callback Function: A function that is passed to another function as an argument and is executed within that function.

    In the example above, dance becomes a callback function when we pass it to toyBox. It’s all about the context in which the function is used!

    High-Order Function

    A function becomes a higher-order function in JavaScript when it meets one of the following criteria:

    • Takes a Function as an Argument: The function accepts another function as an input parameter.
    • Returns a Function: The function produces another function as its output.

    1. Higher-Order Function Taking a Function as an Argument

      Here’s an example where a function takes another function as an argument:

      // Higher-order function
function greetUser(greetingFunction) {
    const userName = "Alice";
    greetingFunction(userName); // Calling the passed function
}

// Callback function
function sayHello(name) {
    console.log("Hello, " + name + "!");
}

// Using the higher-order function
greetUser(sayHello); // sayHello is passed as an argument

    2. Higher-Order Function Returning a Function

      Here’s an example where a function returns another function:

      // Higher-order function
function createMultiplier(multiplier) {
    return function (number) {
        return number * multiplier; // Inner function
    };
}

// Using the higher-order function
const double = createMultiplier(2); // Returns a function that doubles numbers
console.log(double(5)); // Outputs: 10

    Summary

    • A function becomes a higher-order function if it either takes a function as an argument or returns a function.
    • This allows for more flexible and reusable code, enabling you to create more abstract and powerful functionality in your programs.
